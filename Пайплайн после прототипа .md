# Пайплайн задач: Симулятор развития навыков CPO (TypeScript + Vite)

За основу пайплайна принят последний утверждённый HTML-прототип, который фиксирует целевой визуальный baseline MVP и дальнейшую структуру интерфейса.
Vite остаётся корректным инструментом сборки для статического фронтенда с выходом в `dist/`, а парсинг Excel и Word в браузере остаётся совместимым с выбранным стеком через SheetJS и Mammoth.

## Правила пайплайна

- Каждая задача должна завершаться проверяемым результатом.
- Сначала строится фундамент проекта, потом функциональные модули.
- Везде, где возможно, логика должна покрываться типами и проверками на уровне TypeScript.
- Каждая новая задача не должна ломать уже работающий пользовательский сценарий.
- Утверждённый HTML-прототип считается обязательным визуальным ориентиром и не должен произвольно пересобираться в другой UX-сценарий.
- Весь UI-текст и пользовательские сообщения должны быть на русском языке.
- Для хранения данных в рамках MVP запрещено использовать `localStorage` и `sessionStorage`.

## Ключевые корректировки к прошлой версии

- Из пайплайна убираются отдельные полноэкранные этапы `auth` и `home`, потому что текущий утверждённый UI их не использует как самостоятельные состояния.
- Экраны `diagnostic-upload` и `simulator-upload` больше не рассматриваются как отдельные полноэкранные страницы — их логика встраивается в нижние зоны соответствующих разделов.
- Навигация теперь строится вокруг двух верхнеуровневых разделов: `diagnostic` и `simulator`, а все шаги идут внутри одного основного демонстрационного окна.
- Основная задача ранней фазы — не «придумать заново интерфейс», а перенести утверждённый HTML в типизированный TypeScript-проект без потери UX-логики.

# ТЗ-Промт: Симулятор развития навыков CPO в IT-корпорации

## Контекст для AI-агента

Ты — senior full-stack разработчик. Твоя задача — создать веб-приложение **«Симулятор развития навыков CPO»** — интерактивный тренажёр для менеджеров продуктов в крупной IT-корпорации (банковская сфера). Приложение состоит из двух независимых, но связанных разделов. MVP должен работать как статическая HTML-страница в браузере, без серверной части, без интернета (корпоративная интранет-сеть). Все данные загружаются из Excel и Word файлов пользователем вручную через интерфейс.

---

## Стек технологий

- **Frontend:** Vanilla HTML5 + CSS3 + JavaScript (ES6+). Никаких фреймворков (React, Vue и т.д.) — только чистый JS.
- **Файлы данных:** Excel (.xlsx) читается через библиотеку [SheetJS (xlsx)](https://cdn.sheetjs.com/xlsx-latest/package/dist/xlsx.full.min.js) через CDN. Word (.docx) читается через библиотеку [mammoth.js](https://cdn.jsdelivr.net/npm/mammoth/mammoth.browser.min.js) через CDN.
- **AI-интеграция (заглушка на MVP):** Имитация через фиксированные данные из Excel. В коде обязательно выделить отдельную функцию `evaluateWithAI(prompt, criteria)` — пустышку, которая в будущем заменяется на реальный API-вызов (OpenAI-совместимый интерфейс: `POST /v1/chat/completions`).
- **Хранение данных:** Только in-memory (переменные JS). localStorage и sessionStorage не использовать — приложение работает в корпоративном iframe-окружении.
- **Язык интерфейса:** Русский язык.

## Дизайн-система

### Цветовая палитра
```
--color-bg: #f8f8f8            /* светло-серый фон */
--color-surface: #ffffff       /* белые карточки */
--color-text: #1a1a1a          /* почти чёрный */
--color-text-muted: #555555    /* тёмно-серый */
--color-primary: #7c5cbf       /* сиреневый — основной акцент */
--color-primary-hover: #6347a8
--color-success: #3a9e6f       /* зелёный — успех, прогресс */
--color-border: rgba(0,0,0,0.10)
--color-divider: rgba(0,0,0,0.07)
```

### Типографика
- Шрифт: `Inter` (Google Fonts) — основной шрифт для всего интерфейса.
- Заголовки: `Inter`, weight 600–700.
- Тело: `Inter`, weight 400, размер 16px.
- Мелкие подписи: 13–14px, color-text-muted.

### Общие UI-правила
- Скруглённые углы: 8–12px на карточках, 6px на кнопках.
- Тени: мягкие `box-shadow: 0 2px 12px rgba(0,0,0,0.07)`.
- Кнопки: заливка сиреневым для primary, outline для secondary.
- Никаких градиентов на кнопках.
- Прогресс-бар: зелёный `--color-success`.
- Адаптивность: минимум поддержка экранов от 1024px (корпоративные ноутбуки).

## РАЗДЕЛ 1: Оценка навыков (Диагностика)

### Цель раздела
Пользователь проходит диагностику своих продуктовых hard skills на основе кейса. Получает оценки по каждому навыку, комментарии и рекомендации для обучения в Разделе 2.

### Экран 1.0 — Загрузка данных (Административный экран)

> **Важно:** Этот экран — для администратора/фасилитатора, не для конечного пользователя. На MVP он открывается первым при запуске приложения.

**Элементы экрана:**
- Заголовок: «Загрузка данных для диагностики»
- Кнопка «Загрузить Excel с критериями» → принимает `.xlsx` файл
- Кнопка «Загрузить Word с кейсом» → принимает `.docx` файл
- Индикатор успешной загрузки каждого файла (зелёная галочка + название файла)
- Кнопка «Начать диагностику» — активируется только когда оба файла загружены
- Текстовая подсказка о структуре ожидаемых файлов

**Логика чтения Excel (.xlsx):**

Файл содержит **две таблицы на одном листе**:

*Таблица 1 — Задания и критерии* (начинается со строки 1):
| Столбец | Название | Описание |
|---|---|---|
| A | Задание для кейса | Текст задания, который увидит пользователь |
| B | Навыки/Описание индикаторов | Название навыка, который оценивается |
| C | Критерии проверки | Критерии оценки ответа (используются для AI-оценки) |
| D | Комментарий для слайда | Комментарий, который получит пользователь после оценки |

Строк с данными: 11 (по одной на каждое задание).

*Таблица 2 — Правила выставления оценок* (начинается после пустой строки после Таблицы 1):
| Столбец | Название | Описание |
|---|---|---|
| A | Оценка | Числовое значение от 1 до 6 |
| B | Требования | Количество выполненных критериев и комментариев для этой оценки |

**Логика чтения Word (.docx):**
- Файл содержит заголовки формата «Задание 1», «Задание 2» ... «Задание 11»
- Под каждым заголовком — текст описания кейса для этого задания
- Парсить через mammoth.js, извлекать текст по заголовкам
- Связывать с данными Excel по номеру задания

### Экран 1.1 — Введение в кейс

**Элементы экрана:**
- Заголовок: «Диагностика навыков продуктового управления»
- Краткое описание: что будет происходить, сколько заданий, как оценивается
- Блок с общим описанием кейса (вводная часть из Word-документа — первый раздел до Задания 1)
- Кнопка «Начать» → переход к Заданию 1

### Экран 1.2 — Прохождение заданий (11 шагов)

**Элементы каждого шага:**
- Прогресс-бар сверху: «Задание X из 11»
- Название навыка (из столбца B Excel) — мелким текстом над заданием
- Текст задания из Word (заголовок + описание кейса для этого задания)
- Текстовое поле `<textarea>` для ответа пользователя (минимум 4 строки)
- Счётчик символов под полем ввода
- Кнопка «Далее» — активируется при вводе минимум 50 символов
- Кнопка «Назад» — возврат к предыдущему заданию (ответ сохраняется)

**Валидация:** минимум 50 символов в ответе, иначе кнопка «Далее» неактивна с подсказкой.

### Экран 1.3 — Обработка результатов

**Визуальная заглушка AI:**
- После последнего задания — экран «Анализ ваших ответов...»
- Анимированный прогресс-бар (3–5 секунд)
- Текст: «Система анализирует ваши ответы по критериям...»
- После завершения анимации — автоматический переход к результатам

```javascript

  // MVP: возвращает фиксированную оценку из Excel (первое подходящее значение из scoringRules)
```

### Экран 1.4 — Результаты диагностики

**Элементы экрана:**
- Заголовок: «Результаты диагностики»
- Для каждого из 11 заданий — карточка с:
  - Название навыка
  - Оценка (число от 1 до 6) с визуальным индикатором (цветная плашка: 1–2 красный, 3–4 жёлтый, 5–6 зелёный)
  - Комментарий из столбца D Excel
- Итоговый блок «Рекомендации»:
  - Список навыков с низкими оценками (≤3) — «Рекомендуем изучить»
  - Список навыков с высокими оценками (≥4) — «Можете изучить для углубления»

## РАЗДЕЛ 2: Симулятор обучения

### Цель раздела
Пользователь обучается выбранным навыкам через чередование теории и практических кейсов. По каждому навыку получает итоговую оценку.

### Экран 2.0 — Загрузка данных симулятора (Административный экран)

**Элементы экрана:**
- Заголовок: «Загрузка контента симулятора»
- Кнопка «Загрузить Word с теорией и кейсами» → принимает `.docx` файл
- Индикатор загрузки
- Кнопка «Открыть симулятор» — активируется после загрузки

**Структура ожидаемого Word-документа для симулятора:**
```
# Навык: [Название навыка]
## Теория: [Заголовок блока теории]
[Текст теории]
## Кейс: [Заголовок части кейса]
[Описание ситуации]
### Задание
[Текст задания]
### Варианты ответа (опционально)
- [Вариант А]
- [Вариант Б]
- [Вариант В]
### Правильный ответ
[Текст правильного ответа или буква варианта]
### Обратная связь
[Текст который показывается после ответа]
```

> Агент должен распарсить документ по этой структуре и построить объект данных в JS.

### Экран 2.1 — Выбор навыка

**Элементы экрана:**
- Заголовок: «Симулятор развития навыков»
- Подзаголовок: «Выберите навык для изучения»
- Карточки навыков (на MVP — 3–4 навыка):
  - Анализ рынка и конкурентов
  - Работа с гипотезами
  - Видение продукта
  - Метрики продукта
- Каждая карточка содержит: название навыка, краткое описание (1–2 строки), статус прохождения (иконка замка / прогресс / галочка)
- Визуальная плашка «Прогресс будет сохраняться между сессиями (в разработке)» — серым цветом, для демонстрации будущего функционала

### Экран 2.2 — Обучение навыку (основной экран симулятора)

**Принцип работы:**
Чередование блоков «Теория» и «Кейс» согласно структуре из Word-документа. Пользователь движется линейно, блок за блоком.

**Блок «Теория»:**
- Иконка/плашка «Теория» (сиреневый цвет)
- Заголовок блока теории
- Текст теории (полностью, с форматированием из Word)
- Кнопка «Продолжить» → следующий блок

**Блок «Кейс — описание ситуации»:**
- Иконка/плашка «Кейс» (зелёный цвет)
- Описание ситуации
- Кнопка «Перейти к заданию» → показывает задание

**Блок «Задание»:**
- Текст задания
- **Если есть варианты ответа** → показать radio-кнопки с вариантами + кнопка «Ответить»
- **Если нет вариантов** → показать `<textarea>` для текстового ответа + кнопка «Ответить»
- Минимум для текстового ответа: 30 символов

**Блок «Обратная связь»:**
- Показывается после нажатия «Ответить»
- Правильный ответ (или эталонный текст)
- Комментарий/объяснение
- Кнопка «Следующий блок» → продолжение обучения

**Прогресс-бар** сверху показывает: текущий шаг / общее количество шагов по навыку.

### Экран 2.3 — Итоги по навыку

**Элементы экрана:**
- Заголовок: «Навык пройден: [Название навыка]»
- Итоговая оценка (рассчитывается как % правильных ответов на закрытые вопросы)
- Список всех заданий с результатами (правильно / требует доработки)
- Кнопка «Вернуться к выбору навыков» → Экран 2.1
- Кнопка «Вернуться к результатам диагностики» → Раздел 1, Экран 1.4 (если открыт)

## Требования к качеству кода

1. **Модульность:** Разбить JS на логические модули/файлы или чётко выделенные функции с комментариями.
2. **Обработка ошибок:** Все операции с файлами обёрнуты в try/catch. При ошибке парсинга — понятное сообщение пользователю на русском.
3. **Состояние приложения:** Все данные хранить в одном глобальном объекте `AppState = {}` с чёткой структурой.
4. **Комментарии:** Каждая функция имеет JSDoc-комментарий на русском языке.
5. **TODO-метки:** В коде обязательно расставить `// TODO:` метки в местах, где в будущем подключается реальный AI, авторизация, сохранение прогресса.
6. **Единый файл на MVP:** Всё в одном `simulator.html` со встроенными `<style>` и `<script>`.

- Не реализовывать систему уведомлений или email-рассылок
